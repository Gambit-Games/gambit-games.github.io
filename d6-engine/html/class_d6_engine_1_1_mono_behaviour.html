Detailed Description

MonoBehaviour is the base class for components which can be enabled and disabled.

Every high-level script in D6 Engine should derive from MonoBehavior. Unlike Unity, MonoBehaviors inherit directly from the Component class.

◆ Awake()

virtual void D6Engine::MonoBehaviour::Awake ( )
inlinevirtual

Awake is called when the script instance is being loaded. Awake is called either when an active GameObject that contains the script is initialized when a Scene loads, or when a previously inactive GameObject is set to active, or after a GameObject created with Instantiate() is initialized. Use Awake to initialize variables or states before the application starts.

◆ Start()

virtual void D6Engine::MonoBehaviour::Start ( )
inlinevirtual

Start is called on the frame when a script is enabled just before any of the Update methods are called the first time. Like the Awake function, Start is called exactly once in the lifetime of the script.

Start may not be called on the same frame as Awake if the script is not enabled at initialisation time.
